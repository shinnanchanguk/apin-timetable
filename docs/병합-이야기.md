# 압핀 시간표 프로젝트, 왜 이렇게 하나로 합쳤을까요

이 글은 박준일 선생님과 제가 각자 만든 압핀 시간표 프로그램을 하나로 합치면서 어떤 것을 왜 골라 담았는지 개발을 모르는 분도 알 수 있게 풀어 쓴 기록이에요.

## 한 문장 요약

두 선생님이 서로 다른 방식으로 만든 두 프로그램을 각자의 장점만 모아 완성본 하나로 합쳤어요. 기본은 안정적인 쪽으로 두고 다른 방식은 원본을 대조하거나 앱이 못 알려주는 정보를 채우는 데 쓰도록 했어요.

## 무엇을 합쳤나요

압핀 시간표를 프로그램으로 읽는 길이 두 가지 있어요.

- **앱 방식**: 압핀 앱이 하는 것처럼 서버에 "이 학교, 이 반, 이번 주 시간표 주세요"라고 물어보는 길이에요. 제가 이 방식으로 만들었어요.
- **파일 방식**: 학교 폴더에 이미 올라와 있는 시간표 파일을 통째로 내려받아 한 줄씩 읽는 길이에요. 박준일 선생님이 이 방식으로 만드셨어요.

두 방식은 같은 시간표를 보는 서로 다른 문이에요. 그래서 어느 하나만 남기기보다 둘을 한 프로그램 안에 나란히 두고 필요할 때 골라 쓰게 했어요.

## 재료 네 가지와 각각의 좋은 점

이번 완성본에는 네 가지 재료가 들어갔어요.

1. **박준일 선생님의 원래 파일 방식 프로그램**  
   압핀 서버가 시간표 파일을 어떻게 담아 두는지 직접 뜯어보고 알아내신 게 큰 자산이에요. 파일 종류, 한 줄이 어떻게 생겼는지, 글자를 어떤 방식으로 저장하는지까지 꼼꼼히 적어 두셨어요.

2. **제가 만든 앱 방식 프로그램**  
   학교명만 넣으면 바로 시간표가 나오고 결과가 깔끔하게 정리돼 다른 프로그램(쫑알 같은)에 바로 넣기 좋아요. 지역명을 알아서 다듬고 같은 학교를 짧은 시간 안에 다시 물으면 기억해 두는 편의 장치도 있어요.

3. **박준일 선생님이 이번에 고쳐서 다시 올려주신 것**  
   지난번에 두 프로그램을 맞춰 보다가 선생님 설명 글에 파일 종류가 일부 반대로 적혀 있던 걸 함께 찾았어요. 선생님이 그걸 바로잡고 특별실 시간표와 교시 시작시각처럼 앱 방식이 잘 안 알려주는 정보까지 읽도록 다듬어 다시 공유해 주셨어요.

4. **이번에 합친 통합본**  
   위 셋을 하나로 모으면서 두 방식이 정말 같은 결과를 내는지 자동으로 맞춰 보는 기능을 새로 넣었어요.

## 왜 이렇게 합쳤을까요

**두 방식을 다 남긴 이유.** 하나가 막혀도 다른 하나로 확인할 수 있으니까요. 앱 방식이 어느 날 답을 안 주면 파일 방식으로 대신 읽을 수 있고 반대로 파일이 이상하면 앱 방식과 대조해 무엇이 틀렸는지 알 수 있어요.

**기본은 앱 방식으로 둔 이유.** 앱 방식은 학교명만 알면 바로 시작할 수 있고 지금이 몇 주차인지도 날짜만으로 안정적으로 계산돼요. 그래서 평소 시간표를 가져올 때는 앱 방식을 기본으로 두고 파일 방식은 대조와 보강에 쓰도록 했어요.

**두 방식을 자동으로 맞춰 보는 기능을 넣은 이유.** 압핀은 만든 회사가 알아서 운영하고 바꾸는 서버라, 어느 날 형식이 바뀌면 프로그램이 조용히 틀린 값을 줄 수도 있어요. 그래서 같은 반 시간표를 두 길로 각각 가져와 서로 같은지 한 번에 확인하는 기능을 정식으로 넣었어요. 값이 어긋나면 어디가 다른지도 짚어 줘요.

**특별실 시간표와 교시 시작시각은 파일 방식에서 얻는 이유.** 이 두 가지는 앱 방식이 깔끔하게 알려주지 않아요. 박준일 선생님의 파일 방식이 이걸 읽을 수 있어서 통합본에서는 이 정보를 파일 방식으로 채워요.

**애매한 날짜 값으로 막지 않기로 한 이유.** 학교를 찾을 때 서버가 20270220 같은 날짜를 함께 주는데, 이게 구독 끝나는 날인지 다른 뜻인지 확실하지 않아요. 확실하지 않은 값 때문에 멀쩡한 조회를 막으면 안 되니까, 이 값은 참고로만 보여주고 조회를 막는 데는 쓰지 않기로 했어요.

**결과 모양을 한 가지로 맞춘 이유.** 앱 방식이든 파일 방식이든 같은 모양으로 결과를 돌려줘요. 한 결과 안에 요일과 교시로 짜인 표 모양과 한 칸씩 늘어놓은 목록이 모두 들어 있어서 화면에 표로 그리기도 저장하기도 편해요.

## 실제로 확인한 것

대구 학남고등학교로 맞춰서 두 방식을 나란히 돌려 봤어요.

- 학교 코드가 양쪽 모두 0650으로 같았어요.
- 1학년 1반의 월요일부터 금요일까지 모든 시간표가 두 방식에서 똑같이 나왔어요.
- 교사 시간표와 교시 시작시각(08:30, 09:30 이렇게)도 일치했어요.
- 특별실(정보실) 시간표도 파일 방식으로 잘 읽혔어요.

즉, 두 방식은 같은 시간표를 다른 문으로 본 것뿐이고 결과는 같다는 걸 실제 학교로 확인했어요.

## 앞으로 더 볼 것

- 학남고 말고 다른 압핀 학교에서도 파일 종류와 줄 순서가 똑같은지 몇 곳 더 확인하면 좋아요.
- 특별실이 여러 개인 학교나 학년마다 교시 시각이 다른 학교에서는 파일이 어떻게 생겼는지 표본을 더 봐야 해요.
- 압핀은 만든 회사가 언제든 형식을 바꿀 수 있으니, 위의 자동 대조 기능으로 가끔 확인하는 걸 권해요.

## 고맙습니다

이 완성본은 박준일 선생님이 서버 구조를 직접 파고들어 알아내 주신 덕분에 가능했어요. 두 방식을 함께 두고 서로 맞춰 볼 수 있게 된 것도, 선생님이 만든 파일 방식이 있었기 때문이에요.
